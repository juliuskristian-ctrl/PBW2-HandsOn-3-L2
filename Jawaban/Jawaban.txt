Experimen 6 
Perhatikan perbedaan kode antara vulnerable dan secure:

```
// VULNERABLE
container.innerHTML = `<div>${userInput}</div>`;

// SECURE
const div = document.createElement('div');
div.textContent = userInput;
container.appendChild(div);
```

**Pertanyaan:** Jelaskan dengan kata-kata sendiri mengapa versi secure tidak vulnerable terhadap XSS.
 Jawaban    : Versi SECURE aman karena ia memperlakukan input pengguna murni sebagai teks biasa, bukan sebagai kode HTML.
 Pada versi VULNERABLE (innerHTML): Browser diperintahkan untuk "menggambar" apa pun yang ada di dalam variabel, termasuk jika ada tag <img onerror="...">. Browser menganggapnya sebagai perintah HTML yang harus dijalankan.
 Pada versi SECURE (textContent): Browser diperintahkan untuk "menulis" karakter apa adanya. Jika user memasukkan <img ...>, browser tidak akan mencoba menjalankan perintahnya, melainkan hanya menampilkan tulisan <img ...> di layar (seperti mengetik di Notepad).
